# EcomImageGen - 电商 AI 图片生成工具

## 一、项目概述

| 项目信息 | 详情 |
|----------|------|
| 项目名称 | EcomImageGen（电商智图） |
| 项目类型 | 前端 SPA（Vue 3 + Vite）+ 轻量服务端，图片存储在服务端 `gens/` 目录 |
| 目标用户 | 电商卖家、运营人员、电商设计师 |
| 核心价值 | 输入提示词或上传参考图，一键生成可商用的电商商品图 |

---

## 二、技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | Vue 3 + Vite | 现代化构建，开发体验好 |
| UI 组件库 | Ant Design Vue 4.x | 企业级组件库，电商场景成熟 |
| 状态管理 | Pinia | Vue 3 官方推荐，轻量直观 |
| 样式方案 | Tailwind CSS 4 + Less | 原子化 CSS + 主题变量 |
| HTTP 客户端 | Axios | 统一拦截器，便于管理 API 调用 |
| 图片处理 | Canvas API + cropperjs | 前端图片裁剪、编辑 |
| 路由 | Vue Router 4 | SPA 路由管理 |
| 存储 | localStorage + 服务端文件系统 (`gens/`) | 配置存 localStorage，图片文件存服务端 `gens/` 目录，历史记录只存 URL 路径 |
| AI 接口 | OpenAI 兼容格式 | 标准 `/v1/images/generations` 等 |
| PSD 导出 | `ag-psd` | PSD 分层文件生成与导出 |
| PNG 元数据 | `png-metadata` | 写入 PNG 的 metadata（版权、描述等） |
| JPG EXIF | `piexifjs` | 写入 JPG 的 EXIF 信息 |
| ZIP 打包 | `jszip` | 批量下载时打包为 ZIP |
| 图片裁剪 | `cropperjs` | 前端图片裁剪与编辑 |

## 三、核心功能模块

### 3.1 大模型多配置管理（设置页）

用户可配置**多个** AI 大模型，并在生成时自由切换。每个配置独立保存，支持预设厂商模板。

#### 支持的厂商与预设模板

| 厂商 | 预设名称 | API Base URL 模板 | 接口格式 | 说明 |
|------|----------|-------------------|----------|------|
| OpenAI | OpenAI DALL·E | `https://api.openai.com/v1` | OpenAI 标准 | 原生兼容 |
| 阿里云 | 通义千问 / 万相 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | OpenAI 兼容 | 阿里云百炼平台 |
| 腾讯云 | 腾讯混元 | `https://hunyuan.tencentcloudapi.com/` | 自定义 | 需单独适配 |
| 即梦 AI | 即梦生图 | 用户填写 | 自定义 | 需根据实际 API 文档适配 |
| Stable Diffusion | SD WebUI / ComfyUI | `http://localhost:7860/v1` | OpenAI 兼容（需插件）| 本地部署 |
| 自定义 | 自定义 OpenAI 兼容 | 用户填写 | OpenAI 兼容 | 通用兜底 |

> **说明**：代码中使用「适配器模式」，每种厂商类型对应一个 Adapter，负责将统一的前端请求格式转换为对应厂商的实际 API 格式。新增厂商只需新增 Adapter，不影响核心逻辑。

#### 配置管理功能

| 功能 | 说明 |
|------|------|
| 新增配置 | 选择预设厂商模板或自定义，填写参数后保存 |
| 编辑配置 | 修改已有配置的名称、Key、URL、模型等 |
| 删除配置 | 删除不需要的配置（至少保留一个） |
| 切换激活 | 设置某个配置为「当前活跃」，生成页默认使用 |
| 快速切换 | 生成页面顶部显示当前模型名称，点击下拉可切换其他已配置模型 |
| 连接测试 | 保存前/后均可点击「测试连接」按钮，验证 Key 和 URL 是否有效 |
| 导入/导出 | 支持将配置加密导出为 JSON 文件，便于多设备迁移 |

- 配置页面路径：`/settings`
- 配置数据存于 `localStorage`（key: `ecom_ai_profiles`）
- 未配置任何模型时，生成页显示空状态引导用户去配置

### 3.2 文生图（Text2Image）

**路径**：`/generate` 或 `/generate?mode=text2img`

**输入参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| prompt | string | 图片描述提示词（必填） |
| negativePrompt | string | 不希望出现的元素（可选） |
| aspectRatio | string | 图片比例预设：`1:1` / `4:3` / `16:9` / `3:4` / `9:16` / 自定义 |
| resolution | string | 输出分辨率预设：`1K` / `2K` / `4K`（与比例联动，决定最终像素） |
| width | number | 输出图片宽度（根据 ratio + resolution 自动计算，也可自定义覆盖） |
| height | number | 输出图片高度（根据 ratio + resolution 自动计算，也可自定义覆盖） |
| model | string | 使用的模型 ID（从配置读取，可覆盖） |
| batchSize | number | 一次生成张数（1~4） |
| stylePreset | string | 风格预设（电商风/极简/高级感等） |

**分辨率 + 比例组合说明**：

| 分辨率 | 1:1 | 4:3 | 16:9 | 3:4 | 9:16 |
|--------|------|------|-------|------|-------|
| **1K** | 1024×1024 | 1179×884 | 1365×768 | 884×1179 | 768×1365 |
| **2K** | 2048×2048 | 2358×1768 | 2730×1536 | 1768×2358 | 1536×2730 |
| **4K** | 4096×4096 | 4716×3536 | 5460×3072 | 3536×4716 | 3072×5460 |

> 以上为近似像素，实际以模型支持的输出尺寸为准（部分模型只支持固定尺寸，如 DALL·E 3 只支持 1024×1024 / 1792×1024 / 1024×1792）。

**比例预设与电商场景对照**：

| 比例 | 适用场景 |
|------|----------|
| 1:1 | 淘宝/京东/亚马逊 主图（正方形） |
| 4:3 | 详情页宽图、Banner |
| 16:9 | 电脑端横版广告、视频封面 |
| 3:4 | 小红书封面、Instagram 竖版 |
| 9:16 | 抖音/快手 竖版全屏、朋友圈广告 |
| 自定义 | 手动输入宽高比或精确像素 |

> **注意**：部分模型（如 DALL·E 3）有固定输出尺寸，不支持任意比例。此时有两种处理方式：
> 1. 前端超分辨率 + 裁剪：先生成最大支持尺寸，再前端裁剪为目标比例
> 2. 模型支持 `response_format: b64_json` 时，可生成后前端缩放

**交互流程**：
1. 用户在底部输入框输入提示词（支持多语言）
2. 通过快捷栏选择比例 + 分辨率（也可展开高级选项详细设置）
3. 点击「生成」→ 上方结果区显示加载动画
4. 生成完成 → 结果以对话卡片形式展示在上方区域
5. **悬停图片** → 显示操作悬浮层（下载/编辑/收藏/复制Prompt）
6. **点击图片** → 进入全屏预览模式，支持缩放、Before/After对比（图生图）、实际像素查看
7. **右键图片** → 快捷操作菜单（下载PNG/JPG/WebP/复制参数/删除）

**生成状态机（State Machine）**：

```
┌─────────┐    点击生成     ┌──────────┐    API 返回     ┌─────────┐
│  idle   │ ─────────────→ │ loading  │ ─────────────→ │ success │
│ (空闲)  │                │ (生成中)  │                │ (成功)  │
└─────────┘                └────┬─────┘                └────┬────┘
     ↑                          │    API 报错/超时           │
     │                          ↓                            │
     │                    ┌──────────┐                       │
     └─────────────────── │  error   │ ←─────────────────────┘
        用户点击重试       │ (失败)   │      用户点击重试
                         └──────────┘
```

| 状态 | UI 表现 | 用户操作 | 状态流转 |
|------|---------|----------|----------|
| **idle** | 输入框可编辑，生成按钮可用 | 输入提示词、调整参数、点击生成 | → loading |
| **loading** | 结果区底部显示进度条 + 预估时间，输入框置灰，生成按钮变「取消」 | 可点击取消中断请求 | → success / error / idle（取消） |
| **success** | 结果卡片展示在上方区域，显示下载/编辑/收藏按钮 | 悬停/点击/右键操作图片 | → idle（开始新生成） |
| **error** | 结果区显示红色错误卡片（含错误信息 + 重试按钮） | 点击重试、修改参数后重试 | → loading |

> **状态管理**：使用 Pinia Store 统一管理生成状态，确保页面切换后状态不丢失（如从生成页切到历史记录再回来，仍在 loading 则继续显示进度）。

### 3.3 图生图（Image2Image）

**路径**：`/generate?mode=image2img`

**输入参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| referenceImage | file/base64 | 上传的参考图 |
| prompt | string | 生成提示词 |
| negativePrompt | string | 不希望出现的元素（可选） |
| strength | number | 与原图相似度（0~1，越高变化越大，越低越接近原图） |
| width | number | 输出宽度 |
| height | number | 输出高度 |

**交互流程**：
1. 在右侧输入区上方上传参考图（格式：jpg/png/webp，最大 10MB）
2. 上传后预览显示在输入区上方，同时自动将参考图上传到服务端保存到 `gens/YYYY/MM/DD/{recordId}/ref.png`（生成历史记录时 URL 关联引用）
3. 在底部输入框输入提示词，设置相似度滑块（0.1~1.0，默认 0.7）
4. 点击生成 → 上方结果区并排展示原图与生成结果对比
5. 结果可下载/编辑/再生成

### 3.4 图片编辑

**入口**：生成结果点击「编辑」按钮进入编辑页 `/editor/:recordId`

> `:recordId` 为历史记录 ID，编辑页根据该 ID 从服务端 `gens/YYYY/MM/DD/{recordId}/` 目录通过 URL 加载原图进行编辑。

**编辑功能清单**：

| 功能 | 说明 | 实现方式 |
|------|------|----------|
| 裁剪 | 自由裁剪 / 固定比例裁剪（1:1、4:3、16:9 等） | cropperjs |
| 调色 | 亮度、对比度、饱和度、色温调整 | Canvas 滤镜 |
| 加文字 | 添加文字水印/标签，可选字体、颜色、大小、位置 | Canvas + 文字图层 |
| 滤镜 | 预设滤镜（复古、冷调、暖调、黑白、高饱和等） | Canvas 滤镜预设 |
| 局部重绘 | 涂抹区域后重新生成该区域（调用 inpainting 接口） | 需要模型支持，可选实现 |
| 扩图（Outpainting） | 扩展图片边界内容 | 需要模型支持，可选实现 |
| 下载（PNG/JPG/WebP） | 导出为常用图片格式，可选质量 | Canvas.toBlob / a.download |
| **导出 PSD（分层）** | **导出为 PSD 文件，Photoshop 直接打开可编辑** | 见下方 PSD 导出详细说明 |
| **右键菜单（快速操作）** | 生成结果图片支持右键菜单：下载/收藏/复制 Prompt/删除 | 纯前端实现，无需后端 |

#### 图片下载详细交互

生成结果展示在结果区（对话流样式），每条图片支持以下下载方式：

**① 直接下载（单张）**

- 鼠标悬停图片 → 出现「下载」悬浮按钮（右上角）
- 点击 → 根据默认格式下载（可在「设置」中配置默认下载格式：PNG / JPG / WebP）
- 长按 `Alt` + 点击 → 弹出格式选择菜单（PNG / JPG 质量 80~100 / WebP 质量）

**② 右键菜单（快速操作）**

```
┌─────────────────────┐
│ 💾 下载 PNG           │
│ 📷 下载 JPG (质量90)  │
│ 🖼️ 下载 WebP          │
├─────────────────────┤
│ 💝 收藏               │
│ 📋 复制 Prompt        │
│ 📄 复制参数 (JSON)    │
├─────────────────────┤
│ 🗑️ 删除记录           │
└─────────────────────┘
```

**③ 批量下载**

- 勾选多张图片（左上角出现复选框）
- 点击「批量下载」→ 打包为 ZIP（文件名：`ecom-ai-{日期}-{序号}.zip`）
- ZIP 内文件命名：`{prompt前20字}_{width}x{height}_{序号}.png`

**④ 下载格式全局配置（设置页）**

| 配置项 | 选项 | 默认 |
|--------|------|------|
| 默认下载格式 | PNG / JPG / WebP | PNG |
| JPG 质量 | 滑块 1~100 | 92 |
| WebP 质量 | 滑块 1~100 | 88 |
| 是否包含元数据（Prompt） | 是/否（写入 PNG 的 tEXt chunk 或 JPG 的 EXIF） | 是 |
| 下载文件名模板 | `{prompt}_{size}_{date}` 变量可配 | `{prompt}_{size}` |

> **元数据写入**：将生成时的 Prompt、模型名称、时间戳写入图片文件，方便后续追溯。
> 实现方式：PNG 用 `png-metadata` 库，JPG 用 `piexif` 库，纯前端即可完成。

**⑤ 批量导出原图**

从历史记录页选择多条记录，一键打包下载原图（ZIP），或逐张通过 URL 直接下载。

#### PSD 分层导出（重点功能）

导出后的 PSD 文件包含以下图层结构，美工可直接在 Photoshop 中继续编辑：

```
生成的图片.psd
├── 📄 文字图层（可编辑）
│   ├── 标题文字（用户添加的文字水印，可选多图层）
│   └── 标签文字
├── 🖼️ 商品主体图层（透明背景，单独抠图）
├── 🌄 背景图层（原始生成背景 or 纯色/透明）
├── ✨ 特效图层（滤镜效果，可选，以调整图层形式）
└── 📐 参考线/辅助图层（可选）
```

**技术实现方案**：

| 方案 | 说明 | 推荐度 |
|------|------|--------|
| **ag-psd（前端库）** | 纯前端 JS 库，可在浏览器端生成 PSD 文件，无需后端 | ⭐⭐⭐⭐⭐ 推荐 |
| Canvas 导出 + 后端生成 | 前端传图层数据到后端，后端用 `psd-writer` 等库生成 PSD | ⭐⭐⭐ 需后端 |
| 只导出 PNG 分层打包 | 各图层分别导出 PNG，打包为 ZIP，美工手动导入 PS | ⭐⭐ 兜底方案 |

> **推荐方案**：使用 [`ag-psd`](https://github.com/Agamnentzar/ag-psd) 库，它支持在浏览器中**读取和写入 PSD 文件**，纯前端即可实现 PSD 导出，无需后端支持。
> 安装：`npm install ag-psd`

**导出 PSD 的前置要求**：
- 图片编辑时，各操作（加文字、调色、滤镜）需以「图层」为单位管理，不能把所有操作合并到一张 Canvas 上
- 商品主体需要实现「自动抠图」（调用 AI 抠图接口或前端用 `rembg` WASM 版本），才能实现透明背景独立图层
- 若暂无法实现自动抠图，PSD 中商品主体与背景合并为一层，美工需自行抠图（体验打折扣，但可用）

**PSD 导出分模式**（Phase 3 补充）：

| 模式 | 图层结构 | 说明 |
|------|----------|------|
| **快速模式**（默认） | 背景层 + 合并商品层 + 文字层 | 无需抠图，即刻导出，美工需自行抠图 |
| **精细模式** | 背景层 + **透明背景商品独立层** + 文字层 + 调整层 | 调用 AI 抠图接口，耗时较长，但 PSD 可直接用 |

- 精细模式需要配置「抠图 API」（可复用大模型配置里的 Key，或单独配置）
- 若未配置抠图 API，精细模式按钮置灰并提示「请先配置抠图 API」
- `rembg` WASM 方案因包体积过大（~50MB），**不推荐前端使用**，建议走云端抠图 API

**PSD 导出方式**：
- 默认保存到服务端 `gens/YYYY/MM/DD/{recordId}/export.psd`，前端通过 URL 下载
- 同时提供「另存为」功能，浏览器直接下载到本地

### 3.5 批量生成与历史记录

**路径**：`/history`

**功能**：
- 所有生成记录按时间倒序展示（缩略图 + 提示词 + 参数）
- 支持按类型筛选（文生图/图生图）
- 支持搜索（按提示词关键词）
- 支持删除记录（单个/批量）
- 支持重新生成（加载原参数到生成页）
- 生成参数存 `localStorage`（上限 200 条，可配置），图片文件保存到服务端 `gens/` 目录，历史记录中只记录 URL 路径
- 超出记录上限时自动清理最早记录的参数，同时删除服务端 `gens/YYYY/MM/DD/` 下关联的图片文件夹；缩略图长期保留在 `gens/thumbs/YYYY/MM/DD/` 下

**批量生成**：
- 文生图页面支持一次生成 1~4 张变体
- 结果以网格展示，可逐张编辑或批量下载（打包为 ZIP）

**历史记录增强**（Phase 4 补充）：

| 功能 | 说明 |
|------|------|
| **收藏** | 点击 ⭐ 收藏，收藏记录置顶展示；`favorite` 字段已预留 |
| **标签管理** | 为每条记录打标签（如「夏季新品」「主图」「详情页」），按标签筛选 |
| **文件夹/项目** | 创建项目文件夹（如「618 活动」），将相关生成记录归类 |
| **参数导出** | 将某次生成的 prompt + 参数复制为文本 / JSON，便于复用或分享给团队 |
| **重新生成（增强）** | 加载原参数到生成页，且保留原图作为参考（并排展示） |

## 四、页面结构规划（对话式布局）

> **核心设计理念**：参考 Claude 的布局，左侧功能导航 + 右侧主交互区（上方展示结果，下方输入提示词和设置）。

### 页面路由定义

| 路径 | 页面 | 布局风格 |
|------|------|----------|
| `/` | 首页（功能入口 + 使用说明） | 单栏 |
| `/generate` | 主生成页（默认文生图） | 对话式布局（左导航右交互） |
| `/generate?mode=image2img` | 主生成页（图生图模式） | 对话式布局（输入区增加参考图上传） |
| `/editor/:id` | 图片编辑页 | 三栏布局 |
| `/history` | 历史记录 | 三栏布局 |

## 五、数据结构设计（localStorage + 服务端文件系统）

**存储分层策略**：

| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| 模型配置（`ecom_ai_profiles`） | localStorage | 轻量 JSON，< 50KB |
| 历史记录参数（`ecom_ai_history`） | localStorage | 纯文本参数 + 图片 URL 路径 |
| 图片原图 | 服务端 `gens/` 目录 | 按记录 ID 分文件夹存储 |
| 图片缩略图 | 服务端 `gens/thumbs/YYYY/MM/DD/` 目录 | 100×100 小图，长期保留 |

> **为什么用服务端目录？** 浏览器 localStorage 上限仅 5-10MB，单张 1024×1024 PNG 的 base64 约 1-3MB，200 条记录必然超限。将图片保存到服务端 `gens/` 目录（按 `YYYY/MM/DD/{recordId}` 分层），历史记录中只存 URL 路径（如 `/gens/2026/05/21/{recordId}/0.png`），前端通过 `<img src="/gens/...">` 直接访问，既无容量限制，也便于服务端按日期归档、备份和清理。

```
│   └── ...
└── .gitignore                     # 忽略 gens/ 目录（不提交到版本控制）
```

**文件命名规则**：

| 文件类型 | 命名格式 | 示例 URL |
|----------|----------|----------|
| 原图 | `YYYY/MM/DD/{recordId}/{index}.png` | `/gens/2026/05/21/abc123/0.png` |
| 参考图 | `YYYY/MM/DD/{recordId}/ref.png` | `/gens/2026/05/21/abc123/ref.png` |
| 缩略图 | `thumbs/YYYY/MM/DD/{recordId}_{index}.jpg` | `/gens/thumbs/2026/05/21/abc123_0.jpg` |

**读写流程**：

1. **生成完成保存**：
   - Adapter 收到 API 返回的 `url` → 下载为 Blob → 通过后端 API 上传到服务端 → 保存为 `gens/YYYY/MM/DD/{recordId}/{index}.png`
   - Adapter 收到 API 返回的 `b64_json` → base64 解码为 Buffer → 通过后端 API 上传 → 保存为同上路径
   - 同时生成 100×100 缩略图 → 保存为 `gens/thumbs/YYYY/MM/DD/{recordId}_{index}.jpg`
   - 历史记录中 `imageUrls` 存入相对 URL 字符串（如 `/gens/2026/05/21/{recordId}/0.png`）

2. **读取历史记录**：
   - 列表展示时加载 `/gens/thumbs/YYYY/MM/DD/{recordId}_{index}.jpg`（体积小，加载快）
   - 点击大图 / 编辑时加载 `/gens/YYYY/MM/DD/{recordId}/{index}.png`（原图）
   - 文件不存在时显示占位图并提示「图片文件已删除」

3. **图生图参考图**：
   - 用户上传参考图后，上传到服务端保存为 `gens/YYYY/MM/DD/{recordId}/ref.png`
   - 发送给 API 时从服务端读取该文件转 base64
   - 历史记录中 `referenceImageUrl` 记录 URL 路径

**清理策略**：
1. 历史记录达到 200 条上限时，删除最早记录的 localStorage 参数 + 删除服务端 `gens/YYYY/MM/DD/{recordId}/` 整个文件夹
2. `gens/thumbs/YYYY/MM/DD/` 下的缩略图长期保留（单张约 10KB，200 条约 2MB，不占空间）
3. 提供「一键清理所有图片」功能（保留历史记录参数，仅删除服务端图片文件，方便释放磁盘空间）


## 六、AI 接口调用规范（适配器架构）

### 架构设计：适配器模式

前端生成请求使用**统一的数据结构**，由 `Adapter` 层负责转换为各厂商的实际 API 格式：

```
前端统一请求格式
       ↓
  Adapter 选择器（根据 provider 字段）
       ↓       ↓       ↓       ↓
   OpenAI  Aliyun  Tencent  Jimeng  ...
   适配器   适配器   适配器   适配器
       ↓       ↓       ↓       ↓
   实际 API 请求（各厂商真实格式）
```

### 统一请求格式（前端内部）

```js
// 文生图统一格式
{
  "type": "text2img",
  "prompt": "电商商品图：白色无线耳机，极简风格",
  "negativePrompt": "",
  "width": 1024,
  "height": 1024,
  "batchSize": 1,
  "profile": { /* 当前激活的配置 */ }
}

// 图生图统一格式
{
  "type": "image2img",
  "image": "<base64 或 Blob>",
  "prompt": "替换为户外场景",
  "negativePrompt": "",
  "strength": 0.7,
  "width": 1024,
  "height": 1024,
  "profile": { /* 当前激活的配置 */ }
}
```

### 各厂商接口差异说明

#### OpenAI / 通用 OpenAI 兼容
```http
POST {apiBaseUrl}/v1/images/generations
Authorization: Bearer {apiKey}
Content-Type: application/json

{
  "model": "{modelId}",
  "prompt": "...",
  "n": 1,
  "size": "1024x1024",
  "response_format": "url"
}
```

> **返回格式统一处理**：各厂商适配器必须同时支持 `url` 和 `b64_json` 两种 `response_format`：
> - Adapter 只负责 API 调用，返回**标准化结果**（图片 URL 或 base64 字符串数组）
> - 图片下载/解码/上传到服务端 `gens/` 目录由上层统一封装（`api/ai.js` + `utils/fileStorage.js`）处理，Adapter 不感知存储层
> - 上层始终按 `/gens/YYYY/MM/DD/{recordId}/{index}.png` URL 处理图片引用

#### 阿里云 通义万相（OpenAI 兼容模式）
```http
POST https://dashscope.aliyuncs.com/compatible-mode/v1/images/generations
Authorization: Bearer {DASHSCOPE_API_KEY}
Content-Type: application/json

{
  "model": "wanx-v1",
  "prompt": "...",
  "size": "1024x1024",
  "n": 1
}
```
> **注意**：阿里云百炼平台的万相模型同时支持 OpenAI 兼容模式和原生 DashScope 格式，建议优先使用兼容模式（上例），降低适配成本。如需使用原生 DashScope 格式，格式为 `input.prompt` + `parameters.size`（用 `*` 连接）。

#### 即梦 AI（Jimeng）
> 即梦 API 格式需根据具体文档确认，以下为预估格式，实际开发时需对照官方文档调整：
```http
POST {apiBaseUrl}/v1/images/generations
Authorization: Bearer {apiKey}
Content-Type: application/json

{
  "model": "{modelId}",
  "prompt": "...",
  "width": 1024,
  "height": 1024
}
```

#### 腾讯混元（如接入）
> 混元文生图接口为腾讯云 SDK 格式，非 OpenAI 兼容，需单独适配或使用腾讯云 JS SDK。

## 十、补充优化需求

> 以下功能为需求分析阶段补充的优化项，按优先级排列，可在对应实施阶段一并完成。

### 10.1 Prompt 辅助功能（Phase 1 补充）🔴 高优先级

#### 10.1.1 提示词模板库

针对电商场景的预设提示词模板，用户点击即可填入，降低使用门槛：

| 模板分类 | 典型模板示例 |
|----------|--------------|
| 商品主图 | `专业电商商品图，{商品名}，纯白背景，柔和打光，高清细节，8K` |
| 场景图 | `将{商品名}放置在{场景}中，自然光，景深效果，高级感` |
| 极简风 | `极简风格产品摄影，{商品名}，单色背景，阴影，苹果风格` |
| 生活场景 | `{商品名}在日常家居环境中使用场景，温暖光线，真实感` |
| 高级感 | `奢侈品级别产品摄影，{商品名}，黑色丝绒背景，金色光泽，8K高清` |

- 模板支持变量替换（如 `{商品名}` 用户填写后自动替换）
- 模板库存于前端常量，无需后端
- UI 形式：输入框下方展示「常用模板」快捷标签，点击即填入

#### 10.1.2 提示词中译英

- 多数生图模型对英文提示词效果更好
- 用户输入中文后，点击「翻译」按钮（或勾选「自动翻译」），调用翻译接口（可配置翻译 API，或用大模型本身翻译）
- 实际发送给模型的是翻译后的英文 prompt
- 翻译 API 配置项加入「大模型配置」页面（可选）

#### 10.1.3 负面提示词快捷填入

电商常用负面提示词一键填入 negativePrompt：

```
水印, 文字, 签名, 模糊, 低分辨率, 变形, 多余物体, 杂乱背景,
裁剪, 边框, logo, 品牌名, 质量差, 过曝, 欠曝
```

- UI：negativePrompt 输入框下方有「载入电商常用负面词」按钮
- 用户可编辑，不强制

### 10.2 生成错误处理与超时机制（Phase 1 补充）🔴 高优先级

| 场景 | 处理方式 |
|------|----------|
| 生成超时（默认 60s，可配置） | 显示超时提示，提供「重试」按钮 |

| 返回图片为空 | 提示「生成失败，请调整提示词后重试」 |

- 超时时间在「大模型配置」中可配置（默认 60 秒）
- 错误提示统一用 Ant Design Vue 的 `message.error()` 或 `Modal.error()`

### 10.3 图片预览与对比功能（Phase 2 补充）🟡 中优先级

#### Before / After 对比滑块（图生图专用）

```

│  ┌────────┐  │  ┌────────┐   │
```

### 10.6 超分辨率方案调整（Phase 2 修正）🟡 中优先级

当前方案写了前端用 JS 做超分（`super-resolution.js`），但实际有性能问题：

**各方案对比**：

| 方案 | 优点 | 缺点 | 推荐场景 |
|------|------|------|----------|
| **模型自带超分**（万相高清修复） | 质量最好，无需额外开发 | 依赖特定模型 | 🔴 优先使用 |
| **云端超分 API**（调用独立超分服务） | 质量好，前端压力小 | 需要额外 API Key | 🟡 备选 |
| **前端 WASM 超分**（如 Real-ESRGAN WASM 版） | 纯前端，无需后端 | 性能差，1K→2K 需 10~30s，且包体积大（~50MB） | 🟢 仅作兜底 |

**调整后的方案**：
1. **优先**：检测当前模型是否支持超分（如万相），直接调用
2. **备选**：若模型不支持，且用户配置了超分 API，调用云端超分
3. **兜底**：以上都不可用时，提示用户"当前模型不支持 4K，建议使用 2K"，前端不做超分

> 因此 `utils/super-resolution.js` 调整为「超分调用协调器」，负责按优先级选择超分方式，而不是直接做超分算法。

### 10.7 PSD 导出分模式（Phase 3 补充）🟢 低优先级

当前方案写了 PSD 导出，但「自动抠图」依赖（`rembg` WASM 版）有包体积问题，因此 PSD 导出分两种模式：

| 模式 | 图层结构 | 说明 |

| **精细模式** | 背景层 + **透明背景商品独立层** + 文字层 + 调整层 | 调用 AI 抠图接口，耗时较长，但 PSD 可直接用 |
